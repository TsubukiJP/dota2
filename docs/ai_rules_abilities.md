# Tooltip翻訳ルール

## A. 文体と構文

### 1. 能動態と主体
*   **能動態を原則とする**: ゲームのアビリティやアイテムの効果は、それが何をするかを能動的に記述する。
    *   ×: 敵はダメージを受ける -> 敵にダメージを与える
    *   ×: スタンを受ける -> スタンさせる
    *   ○: 敵に100ダメージを与え、2秒間スタンさせる。
*   **主語の省略**: 特に `modifier_Description` において、「ユニットは〜」などの主語は省略し、状態そのものを記述する。
    *   ×: ユニットはアーマーが-X低下している
    *   ○: アーマーがX低下している。
    *   **受動・状態の統一**: 「〜を与える」「〜を受ける」ではなく、状態を表す「〜している」「〜低下している」等の表現に統一する。
    *   ×: Axeに%val%%%減少したダメージを与える。
    *   ○: Axeに与えるダメージが%val%%%減少している。

### 2. 増減・数値の表現
*   **統一フォーマット**:
    *   「●●をxx減少/増加させる」
    *   「●●がxx増加/低下する」
*   **マイナス記号の扱い**: 原文の `-%variable%` のようにマイナス記号がついている場合、訳文で「減少/低下」を用いるならマイナス記号は省き、二重表現を避ける（文脈による）。
    *   例: `-%val% armor` -> `-%val% アーマー` または `%val% アーマー減少`
    *   例: `-{s:bonus}秒 Eclipse クールダウン` -> `-{s:bonus}秒 Eclipse クールダウン`
*   **単位の翻訳**: 変数に直結する単位（`s`など）は適切に翻訳する。
    *   例: `{s:value}s` -> `{s:value}秒`
    *   例: `%value%s` -> `%value%秒`
*   **スペースの除去**: **説明文（*_Description）中の**変数・数値また％記号と、日本語の単位や助数詞の間にはスペースを入れない（詰める）。（タレントなどのテキスト中は除く）
    *   ×: `+%hp_regen%%% 体力`
    *   ○: `+%hp_regen%%%体力`

### 3. 説明文・注釈
*   対象: `*_Description`, `*_Note*`, `*_note*`
*   **常体（だ・である調）**を使用する。
*   簡潔さを最優先する。

### 4. 状態変化
*   対象: `modifier_*_Description`
*   **現在進行形**で状態を表す（〜している / 増加している / 低下している）。「〜した」は避ける。
*   **Slowの具体的な記述**: 「スローしている」「スローされている」のような直訳を避け、「移動速度が〜減少している」「移動速度が〜低下している」のように具体的に記述する。
    *   ×: 「%value%%%スローしている」
    *   ○: 「移動速度が%value%%%減少している」

### 5. 世界観
*   対象: `*_Lore`
*   Dota 2 のファンタジー世界観を尊重し、重厚かつ自然な文体で翻訳する。
*   直訳調を避け、物語として成立する日本語を目指す。
*   **Dota 2 世界固有の固有名詞は英語維持**:
    *   キャラクター名（ヒーローの本名、非ヒーローキャラ）: `N'aix`, `Selemene`, `Barathrum`
    *   地名: `Outlands`, `Nightsilver Woods`, `Weeping Rose`
    *   種族名・その他: `Keen`, `Oglodi`, `Fundamentals`

## B. 自然なゲーム表現の原則
すべてを網羅することは不可能だが、以下の**「戦闘・競争的な記述原則」**を指針とすること。

*   **原則1：日常語の排除**
    *   日常的な言葉（助ける、受け取る、起こる）を避け、ゲーム・軍事的な言葉（支援、獲得、発生）を選択する。
*   **原則2：主体を「アビリティ/効果」に置く**
    *   「ユニットが〜を受ける」ではなく「効果がユニットに〜を負わせる」または「〜を付与する」と表現する。
*   **原則3：人格と主観の排除（代名詞の排除）**
    *   「彼/彼女/彼ら (him/her/them)」などの代名詞は使用せず、ヒーロー名を明記するか省略する。
    *   「喜ぶ」「悲しむ」「助ける」などの主観的・感情的な表現を排除し（Lore等を除く）、メカニクスとして記述する。

### 特定の禁止・推奨表現
| 英語原文 | 禁止（直訳） | 推奨例（ゲーム表現） |
| :--- | :--- | :--- |
| **take / receive / get** | 受ける、受け取る、もらう | **獲得する、負う（ダメージ等）、得る** |
| **deal / inflict / apply** | 適用する、配る | **与える、負わせる、付与する** |
| **dismount / dislodge** | 降りる、取り外される | **振り落とされる** |
| **the bolt / the unit** | そのボルト、そのユニット | **（文脈で自明なら省略、または名詞を繰り返す）** |
| **... if it kills** | それがキルしたら | **ユニットをキルした場合、トドメを刺した場合** |
