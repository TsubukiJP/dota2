# UI翻訳ルール
Tooltip系以外のUI/システムテキスト専用ルール。

## A. UIデフォルトルール

### 1. 簡潔さと視認性の優先
*   **「短さ」を最優先**: ボタンやラベル、短いメッセージは極限まで簡潔にする。
*   **体言止めの活用**: ラベルや見出しは「〜する」ではなく名詞で終える。
    *   例: `DOTA_Move` → `移動` (×移動する)
    *   例: `DOTA_Shop` → `ショップ`

### 2. 文体の柔軟性
*   **丁寧語（です・ます調）を許容**:
    *   設定説明やユーザーへの案内系テキストは丁寧語が自然な場合がある。
    *   ただし、冗長な「〜することができます」は避け、「〜可能」「〜できる」とする。
*   **感嘆符・疑問符の維持**:
    *   チャットホイールや定型句では `！` `？` を自然に使用してよい。

### 3. 変数を含むメッセージ
*   変数 (`%s1`, `%s2` 等) を含むシステムメッセージは、日本語の語順に自然に並べ替える。
*   ただし、**変数の個数・順序を維持**すること。

### 4. チャット/イベントメッセージ
*   `DOTA_Chat_*` などのゲーム内メッセージは、臨場感を損なわない自然な日本語にする。
*   過度に硬い表現や、逆にカジュアルすぎる表現は避ける。
